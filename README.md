### Exploits
Exploits sind im Generellen eine Sicherheitslücke. Diese Schwachstellen können überall im System liegen, aber auch beim Ausführen von Programmen. Angreifer suchen diese Exploits und nutzen diese dann aus. Man kann verschiedene Programme installieren, welche die Exploits finden sollten.
Es gibt verschiedene Arten, wie man Exploit nutzen kann. Remote ausgeführte Exploits, Lokale ausgeführte Exploits, SQL-Injection-Exploits

Remote-Exploits: Hier zielt man auf die Schwachstellen der Netzwerksoftware und schickt manipulierte Datenpakete für den Angriff.

Lokaler Exploit: Hier spricht man von einem Programmcode, der in einer Datei mit drin ist. Es sind meist harmlose Dateien wie eine Textdatei. Doch beim Öffnen wird sie sofort aktiviert und durchgeführt, ohne das Wissen vom User.

SQL-Injection-Exploits: Hier werden Webanwendungen angegriffen. Man schreibt einen Schadcode für diese SQL-Datenbank.


### Zero-Day Exploit

#### Einführung

Ein Zero-Day-Exploit ist ein Cyberangriff Vektor, der eine unbekannte oder nicht behandelte Sicherheitslücke in Computersoftware, Hardware oder Firmware ausnutzt. "Zero day" bezieht sich darauf, dass der Software- oder Gerätehersteller null Tage Zeit hat, um die Schwachstelle zu beheben, weil der Angreifer sie bereits nutzen kann, um auf gefährdete Systeme zuzugreifen.

#### Wie funktionierts

Die Funktionsweise eines Zero-Day-Exploits hängt von der Art der Sicherheitslücke ab. Ein einzelner Angriff kann auch mehrere Sicherheitslücken gleichzeitig ausnutzen, um einen einzigen Zero-Day so effektiv wie möglich einzusetzen. 

Der Prozess für die Ausnutzung eines Zero-Day beginnt, sobald ein Angreifer die Sicherheitslücke entdeckt hat. Diese Schwachstelle kann sich in Hardware, Firmware, Software oder jedem anderen Unternehmensnetzwerk befinden. Die folgenden Schritte beschreiben einen typischen Zero-Day-Angriff:

1. Entwickler veröffentlichen eine Anwendung oder ein Update für eine Anwendung, das eine bisher unbekannte Sicherheitslücke enthält.

2. Ein Angreifer durchsucht die Software und entdeckt die Sicherheitslücke entweder durch Scannen oder durch das Herunterladen des Codes aus einem Repository und anschließende Analyse.

3. Der Angreifer nutzt Tools und Ressourcen, um die Schwachstelle auszunutzen. Dies kann benutzerdefinierter Code sein, den der Angreifer schreibt, oder bereits vorhandene Tools.

4. Die Sicherheitslücke wird möglicherweise über Jahre hinweg ausgenutzt, bevor sie erkannt wird. Schließlich können Forscher, die Öffentlichkeit oder IT-Experten auf Hacker-Aktivitäten stoßen und die Schwachstelle an die Entwickler melden.

Der Begriff "Zero-Day" bezieht sich auf die Zeit, die Entwickler haben, um eine Sicherheitslücke zu schließen. Wenn der Angreifer die Lücke entdeckt, haben die Entwickler null Tage Zeit, um sie zu beheben. Sobald ein Patch verfügbar ist, gilt die Lücke nicht mehr als Zero-Day. Allerdings können Sicherheitslücken weiterhin bestehen, wenn Administratoren und Benutzer nicht die neuesten Updates installieren. Solche Systeme sind ein häufiger Grund für kritische Daten Verletzungen. Zum Beispiel wurde der Equifax-Hack, bei dem Hacker hunderte Millionen Datensätze stahlen, von einem öffentlich zugänglichen Server verursacht, auf dem die neuesten Patches nicht installiert waren.


#### Ausnutzung und Angriffsmethoden

1. Pufferüberlauf (Buffer Overflow): Bei dieser Technik überschreitet ein Angreifer den für eine Anwendung reservierten Speicherplatz und injiziert schädlichen Code, um die Ausführung zu manipulieren oder die Kontrolle über das System zu übernehmen.

2. Code Injection: Hier injiziert der Angreifer bösartigen Code in die laufende Anwendung, um sie zu kompromittieren. Dies kann beispielsweise durch SQL-Injection, Cross-Site-Scripting (XSS) oder Remote-Code-Execution (RCE) erfolgen.

3. Drive-by-Download: Diese Methode nutzt Sicherheitslücken in Webbrowsern oder anderen Softwareanwendungen aus, um automatisch Malware auf das System eines Benutzers herunterzuladen und auszuführen, wenn dieser eine infizierte Website besucht oder auf einen manipulierten Link klickt.

4. Zero-Day-Exploits für Betriebssysteme: Hierbei werden Schwachstellen in Betriebssystemen wie Windows, macOS oder Linux ausgenutzt, um Zugriff auf das System zu erlangen, Berechtigungen zu erhöhen oder andere schädliche Aktivitäten auszuführen.

5. Firmware-Exploits: Angreifer können auch Sicherheitslücken in der Firmware von Hardwaregeräten wie Routern, Kameras oder IoT-Geräten ausnutzen, um Kontrolle über diese Geräte zu erlangen oder sie als Eintrittspunkt in ein größeres Netzwerk zu verwenden.

6. Social Engineering: Manchmal können Zero-Day-Exploits in Verbindung mit Social-Engineering-Techniken eingesetzt werden, um Benutzer dazu zu verleiten, schädliche Dateien herunterzuladen oder auf Links zu klicken, die die Sicherheitslücke ausnutzen.

Beispiele für Zero-Day Exploits:

Stuxnet war ein raffinierter Computerwurm, der vier verschiedene Zero-Day-Software-Sicherheitslücken in Microsoft Windows-Betriebssystemen ausnutzte. Im Jahr 2010 wurde Stuxnet in einer Serie von Angriffen auf Kernkraftwerke im Iran eingesetzt. Sobald der Wurm die Computersysteme eines Kernkraftwerks infiltriert hatte, sandte er bösartige Befehle an die Zentrifugen, die zur Anreicherung von Uran verwendet wurden. Diese Befehle veranlassten die Zentrifugen, sich so schnell zu drehen, dass sie versagten. Insgesamt wurden 1.000 Zentrifugen durch Stuxnet beschädigt.

Log4Shell war eine Zero-Day-Schwachstelle in Log4J, einer Open-Source-Java-Bibliothek, die zur Protokollierung von Fehlermeldungen verwendet wird. Hacker konnten die Log4Shell-Lücke verwenden, um praktisch jedes Gerät fernzusteuern, das Java-Apps ausführt. Da Log4J in beliebten Programmen wie Apple iCloud und Minecraft verwendet wird, waren Hunderte von Millionen Geräten gefährdet. Die Common Vulnerabilities and Exposures (CVE) Datenbank des MITRE-Instituts verlieh Log4Shell die höchstmögliche Risikobewertung, eine 10 von 10.

Die Log4Shell-Lücke bestand seit 2013, aber Hacker begannen erst 2021, sie auszunutzen. Die Schwachstelle wurde kurz nach der Entdeckung gepatcht, aber Sicherheitsforscher entdeckten, dass während des Höhepunkts mehr als 100 Log4Shell-Angriffe pro Minute stattfanden.

2022 Chrome-Angriffe
Anfang 2022 nutzten nordkoreanische Hacker eine Zero-Day-Schwachstelle zur Remote-Code-Ausführung in Google Chrome-Webbrowsern aus. Die Hacker verwendeten Phishing-E-Mails, um Opfer auf gefälschten Websites zu locken, die die Chrome-Schwachstelle nutzen, um Spyware und ferngesteuerte Zugriffs-Malware auf den Geräten der Opfer zu installieren. Die Schwachstelle wurde umgehend gepatcht, aber die Hacker verwischten ihre Spuren gut, und Forscher wissen nicht genau, welche Daten gestohlen wurden.

### Man-in-the-Middle

#### Einführung

Man-in-the-Middle-Angriffe (MITM) sind eine gängige Art eines Cyber-Angriffs, mit dem Angreifer sich in die Kommunikation zwischen zwei Zielpersonen einschalten. Der Angriff tritt zwischen zwei kommunizierenden Hosts auf, welche es dem Angreifer ermöglichen, eine Unterhaltung mitzuhören, die für ihn normalerweise nicht zugänglich sein sollte, daher der Name „Man-in-the-Middle“.

#### Wie funktionierts

Ein Man-in-the-Middle-Angriff funktioniert, indem der Angreifer entweder physisch oder durch Ausnutzung von Schwachstellen im Netzwerk, wie beispielsweise durch Spoofing-Techniken, zwischen die Kommunikationspartner tritt. Sobald der Angreifer erfolgreich zwischen den Partnern positioniert ist, kann er den Datenverkehr abfangen, überwachen und sogar manipulieren, ohne dass die beteiligten Parteien davon wissen. Der Angreifer kann dann die übertragenen Daten lesen, modifizieren oder sogar neue Datenpakete in die Kommunikation einschleusen, was es ihm ermöglicht, sensible Informationen abzugreifen oder die Kommunikation zu stören. Dies kann durch verschiedene Methoden wie ARP-Spoofing, DNS-Spoofing oder SSL-Stripping erreicht werden, je nach den spezifischen Schwachstellen des Netzwerks und der eingesetzten Sicherheitsmaßnahmen.

#### Ausnutzung und Angriffsmethoden

LAN Angriffe

- physische zugang zu den datenleitung

- Im ethernet die *ARP-tabelle der opfersystem modifizieren durch *ARP-Spoofing. 

 -> ermöglicht das Abhören des Datenverkehrs an Switches. Dies funktioniert, wenn der angreifer in die gleiche Lokalnetz ist. 

-  Der Angreifer hängt am gleichen Netzwerkbus wie das Opfer, das funktioniert nur mit netzwerken mit busstruktur, wie z.B Ethernet mit Hub oder 10base2 sowie bei einer Token-ring-infrastruktur. diese techniken sind sehr veraltet und kommen sehr selten vor.

- Vorspielen eines falschen DHCP-Servers. Durch die Angabe von einer falschen Standard-Gateway-Adresse zum Internet kann die Kommunikation durch einen Rechner des Angreifers geleitet werden. 

WLAN Angriffe

- Bei öffentliche WLAN-hotspots das vortäuschen eines falschen WLAN-access point 	durch *Snarfing. In diesem Fall werden nach Auswertung und Manipulation die Daten zum korrekten Access-Point weitergeleitet.

Weitere MITM-Angriffe

- Der Angreifer hat Kontrolle über einen Router, der durch den Datenverkehr geschleust wird. Dies funktioniert im WAN, LAN und WLAN.

- Durch *DNS Cache Poisoning gibt dem Angreifer eine falsche Zieladresse für die Internetkommunikation vor und leitet den Verkehr zu seinem eigenen Rechner (Poison Routing).

- Durch Manipulation der *Host-Datei auf dem Rechner des Angegriffenen können trotz der Eingabe der echten URL gefälschte IP-Adresse aufgelöst werden.

Bekannte Computerprogramme für MITM-Angriffe sind Ettercap und Cain & Abel.

Begriffe:

ARP Tabelle: Protokoll der Hardware-Adressen(IPv4-Adressen, Mac-Adressen) der Netzzugangsschicht ermittelt.

ARP Spoofing(ARP Request Poisoning): Anfrage an ARP verfälschen oder gefälschte ARP-Paketen senden. wird benutzt, um ein ARP-Tabelle in einem Netzwerk zu ändern und anschließend die Kommunikation in diesem Netzwerk zu manipulieren.

Snarfing: ist Informationsdiebstahl oder Datenmanipulation in kabellosen lokalen Netzwerken wie z.B. WLANs oder WPANs.

DNS Cache Poisoning: Angriff auf DNS-Server. dadurch werden die Zuordnung zwischen einem Domainname und der zugehörigen IP-Adresse gefälscht.

Host-Datei: eine lokale Konfigurationsdatei, um Hostname zu IP-Adressen zuzuordnen. Diese System-Datei wurde ursprünglich an Stelle des DNS verwendet.

### Session Hijacking

#### Einführung 

Beim Session-Hijacking übernimmt der Angreifer die Sitzung von einem Benutzer. Das Ziel ist es, sich in den Account des Benutzers einzuloggen, damit man Zugriff auf die Daten des Benutzers haben kann. Mit den richtigen Berechtigungen kann der Angreifer auf den Domain und Server zugreifen. Dabei sind Daten auf dem Server gefährdet. Es kann zu einer Data Leakage kommen oder Blackmailing mit den Daten. Dies kann Privatnutzer, Personen des öffentlichen Lebens oder Unternehmen treffen. Session Hijacking wird meistens mit Hilfe eines Exploits durchgeführt. Natürlich ist Brute Force auch eine Art von Methode, auch wenn es sich nur um ein einfaches Passwort handelt.

#### Wie funktionierts

Entführung von TCP-Sitzungen
Es wird eine TCP-Verbindung zwischen dem rechtmäßigen Benutzer und dem Server. Der Angreifer versucht nach der Authentifizierung den Austausch zu übernehmen, indem er sich als Benutzer ausgibt. Die Pakete vom Angreifer sind manipuliert und schneller. Dafür braucht der Angreifer jedoch die Sequenznummer. Falls der Austausch verschlüsselt ist, kann die Sequenznummer direkt ausgelesen werden. Wenn die gefälschten Pakete nun vor den echten Paketen ankommen, wird der Server diese auswerten und die echten Pakete ignorieren.
Entführungen von Web-Sitzungen
Beim HTTP wird jede HTTP-Anfrage als neue Verbindung entgegengenommen, bearbeitet und geschlossen. Benutzer von Webanwendungen nutzen andauernd diese HTTP-Anfragen, deswegen bekommen die Benutzer eine Sitzungs-ID generiert. Diese wird vom Browser vom Benutzer bei den Anfragen übermittelt, um sich beim Server zu identifizieren. Meistens wird es mit einem Cookie übermittelt. Wenn ein Angreifer einen Weg findet, diese Sitzungs-ID mitzulesen, kann er sich als den authentifizierten Benutzer ausgeben und die Sitzung übernehmen. Deswegen ist es wichtig, dass man zur Änderung vom Passwort das alte Passwort eingeben muss. Ansonsten kann der Angreifer den Benutzer komplett von der Session ausschließen.

#### Ausnutzung und Angriffsmethoden

Malware-Injektion
Wenn ein Programm mit einem Code erweitert wird, ist das eine Malware-Injektion. Je nach dem betroffenen Programm kann es eine neue Applikation öffnen ohne das Wissen des Benutzers. Unter anderem kann es auch Cookies abfangen und dem Angreifer senden oder persönliche Angaben aus dem Gerät auslesen. Bei einer *SQL-Injektion, bei dem das Programm der Datenbank manipuliert wird, dem Angreifer Adminrechte geben.
Session-Seitenaufbau
Es wird abgezielt, auf den Netzwerkverkehr des Benutzers zuzugreifen. Dies wird mit einem unsicheren Netzwerk erleichtert. Mit *Sniffing greift man öffentliches WLAN oder Websites ohne *SSL-Zertifikate an.
Sitzung Fixierung 
Bei der Sitzungsfixierungs wird dem Benutzer eine Sitzungs-ID vom Angreifer zugestellt. Das kann mit einem einfachen Link, der die Sitzungs-ID ersetzt, geschehen. 

Begriffe:

*SQL: Structured Query Language

*Sniffing:  Der Pakete Austausch im Netzwerk wird überwacht und abgefangen.

*SSL-Zertifikat: Das Secure Socket Layer- Zertifikat verschlüsselt den Datenaustausch.


#### Zukünftige Herausforderungen und Entwicklungen

In Zukunft arbeitet man daran, mit der Hilfe von maschinellem Lernen(KAI) Session Hijacking in Echtzeit zu erkennen, noch bevor irgendein Schaden entstehen kann. Mit der Überwachung von Netzwerkverkehr könnte man schon im voraus Versuche von gefälschten Paketen abfangen. Dann kommt es gar nicht zu einem Session Hijacking. Das Verhalten vom Benutzer könnte die KI analysieren und feststellen, ob sich der Benutzer untypisch verhält. Generell könnte die KI mögliche Angriffe vorausbestimmen und für diese kann man dann vorbeugende Massnahmen treffen. 

In der Offensive gibt es auch Möglichkeiten, um  die KI zu nutzen.  Die KI könnte einfache Schwachstellen ausfindig machen. Mit dem Lernen von vorherigen Angriffen lernt es die beste Angriffsmethode oder die häufigsten Schwachstellen für das Session-Hijacking.

### Verteidigung und Prävention von Exploits

#### Verschlüsselung:

Verschlüsselung ist eine der effektivsten Methoden, um Man-in-the-Middle-Angriffe zu verhindern. Verschlüsselung schützt Ihre Kommunikation davor, von einem Angreifer abgefangen und gelesen zu werden. Verschlüsselung ist der Prozess, bei dem Klartext in Geheimtext umgewandelt wird, um sensible Daten vor unbefugtem Zugriff zu schützen und deren Lesbarkeit durch unbefugte Parteien zu verhindern. Verwenden Sie verschlüsselte E-Mail- und Messaging-Dienste und verwenden Sie immer HTTPS für das Web-Browsing. Viele Websites nutzen mittlerweile standardmäßig HTTPS, was eine zusätzliche Sicherheitsebene bietet. Wenn Sie öffentliche WLANs verwenden, nutzen Sie ein virtuelles privates Netzwerk (VPN), um Ihre Kommunikation zu verschlüsseln.

#### SSL/TLS-Zertifikate überprüfen:

Beim Besuch einer Website sollten Sie überprüfen, ob sie ein gültiges SSL/TLS-Zertifikat hat. SSL/TLS-Zertifikate bieten Verschlüsselung und Authentifizierung für Websites. Wenn eine Website kein gültiges Zertifikat hat, könnte dies ein Anzeichen für einen Man-in-the-Middle-Angriff sein. Suchen Sie nach einem Schlosssymbol in der Adressleiste Ihres Webbrowsers, das darauf hinweist, dass die Website ein gültiges SSL/TLS-Zertifikat hat.

#### sichere WLAN-Netzwerke verwenden:

Vermeiden Sie es, sich mit ungesicherten WLAN-Netzwerken zu verbinden, insbesondere an öffentlichen Orten wie Cafés, Flughäfen oder Hotels. Ungesicherte WLAN-Netzwerke sind anfällig für Man-in-the-Middle-Angriffe, da jeder die Kommunikation zwischen den Geräten, die mit dem Netzwerk verbunden sind, abfangen kann. Verwenden Sie stattdessen ein Mobilfunknetz oder ein vertrauenswürdiges WLAN-Netzwerk, das ein Passwort erfordert. Wenn Sie öffentliche WLAN-Netzwerke nutzen müssen, verwenden Sie ein VPN, um Ihre Kommunikation zu verschlüsseln.
Software auf dem neuesten Stand halten:
Es ist wichtig, Ihre Software auf dem neuesten Stand zu halten, einschließlich Ihres Betriebssystems, Webbrowsers und anderer Software. Software-Updates enthalten oft Sicherheitspatches, die Sicherheitslücken beheben, die bei Man-in-the-Middle-Angriffen ausgenutzt werden können. Stellen Sie Ihre Software so ein, dass sie automatisch aktualisiert wird, oder überprüfen Sie regelmäßig auf Updates.

#### Vorsicht vor Phishing-E-Mails:

Phishing-E-Mails sind eine häufige Methode, mit dem Angreifer Man-in-the-Middle-Angriffe durchzuführen. Phishing-E-Mails sind darauf ausgelegt, wie legitime E-Mails von einer vertrauenswürdigen Quelle wie einer Bank oder einer E-Commerce-Website auszusehen. Sie enthalten oft einen Link zu einer gefälschten Website, die der echten ähnelt. Wenn Sie Ihre Anmeldeinformationen oder andere sensible Informationen eingeben, kann der Angreifer sie abfangen. Seien Sie vorsichtig bei E-Mails von unbekannten Absendern und überprüfen Sie immer die Echtheit von E-Mails, bevor Sie auf Links klicken.

#### Verteidigung und Prävention von Zero-Day Exploit

1. Sicherheits Audits und Penetrationstests: Führen Sie regelmäßige Sicherheitsaudits und Penetrationstests durch, um Schwachstellen in Ihrer Infrastruktur zu identifizieren und zu beheben, bevor sie von Angreifern ausgenutzt werden können.

2. Einschränkung von Berechtigungen: Begrenzen Sie die Berechtigungen von Benutzern und Anwendungen auf das erforderliche Minimum, um das Ausmaß möglicher Schäden bei einem erfolgreichen Angriff zu verringern.

3. Monitoring und Incident Response: Implementieren Sie ein robustes Überwachungssystem, um verdächtige Aktivitäten zu erkennen, und entwickeln Sie ein umfassendes Incident-Response-Verfahren, um schnell auf Sicherheitsvorfälle zu reagieren und sie einzudämmen.
